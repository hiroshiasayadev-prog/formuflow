# 05 — タブバー＋ページ遷移 仕様

対応モック: `docs/mockups/05-tabs-navigation`

---

## タブバー

### 寸法・配置

- 高さ: `35px`、固定
- メインエリアの最上部（メニューバー・アイコンバーより下）

### タブの挙動

VSCode準拠。

- **横スクロール**: タブが多くなったら1列横スクロール（折り返しなし）
- **アクティブタブ**: スクロール位置が自動調整され常に表示される
- **タブ幅**: 可変・最小幅あり（タイトルが省略される: `text-overflow: ellipsis`）
- **`×` ボタン**: hover時のみ表示
- **未保存表示**: draft状態のタブはタイトル左に `●` を表示

### タブで開くページ一覧

| ページ種別 | パス | アイコン | トップライン色 |
|---|---|---|---|
| Flowキャンバス | `/flows/:id` | `⇢` | `#14b8a6` |
| Formula編集 | `/formulas/:id` | `fx` | `#6366f1` |
| DatabaseTable設定 | `/dbtables/:id` | `DB` | `#f43f5e` |
| DefaultInput設定 | `/inputs/:id` | `▶` | `#06b6d4` |
| Const設定 | `/consts/:id` | `C` | `#f59e0b` |
| Consts設定 | `/consts-map/:id` | `C` | `#f59e0b` |

- 全Componentはタブで開いて編集する。インライン編集・右パネル編集は採用しない（設計の一貫性を優先）
- **アクティブタブ上端のトップライン（1px）はComponent種別カラーで表示する**（accent色固定ではない）

### DefaultReturnComponentの扱い

- Flow固有（ツリーに独立エントリとして存在しない）
- **専用の編集ページなし・右パネルも使わない**
- キャンバス上の操作で完結: Returnノードのポートにedgeを繋ぐと、edge元のtitle/変数名がReturnのport名に自動入力される
- port追加の操作は2方向どちらもOK:
  - Returnノードの `+ add` ハンドルからドラッグして他ノードのoutputに繋ぐ
  - 他ノードのoutputハンドルからドラッグしてReturnノードに繋ぐ（通常のedge接続と同じ）

## ページ遷移（タブを開くトリガー）

### Componentツリーから
- ダブルクリック → 該当Componentのページをタブで開く

### Flowキャンバス上から
- ノードを**ダブルクリック** → 該当Componentのページをタブで開く
- ノードの **✏ 編集アイコン** クリック → 同上（ホバー時に表示）

---

## タブの開き方ルール

### 重複抑制（VSCode準拠）
- 同一IDのページが既にタブで開いている場合 → **そのタブをアクティブにする**（新規タブは開かない）
- 別IDの同種ページ → 新しいタブを追加

### 新規タブの挿入位置
- **アクティブタブの右隣**に追加

### タブを閉じる
- `×` ボタンクリック（hover時に表示）
- 閉じた後のアクティブタブ: VSCode準拠（直前にアクティブだったタブ、なければ左隣）
